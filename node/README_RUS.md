# Smart Dacha — Прошивка сенсорного узла

Прошивка для Wemos D1 Mini (ESP8266). Каждый узел считывает температуру и влажность с датчика DHT22 и предоставляет данные по HTTP. Часть системы мониторинга [Smart Dacha](../).

## Оборудование

| Компонент | Спецификация |
|-----------|-------------|
| МК | Wemos D1 Mini (ESP8266) |
| Датчик | DHT22 (AM2302) — температура и влажность |
| Пин датчика | `D6` (настраивается в `config.h`) |

Развёрнуто три идентичных узла, по одному на комнату:

| Комната | Статический IP |
|---------|----------------|
| Спальня (Bedroom) | `192.168.0.101` |
| Кухня (Kitchen) | `192.168.0.102` |
| Гостиная (Living Room) | `192.168.0.103` |

### Последовательность загрузки

1. Инициализация последовательного порта на 115200 бод
2. Запуск датчика DHT22 (задержка 2 сек. для стабилизации)
3. Настройка WiFi в режиме станции (STA) со статическим IP
4. Подключение к сети WiFi (блокирующее ожидание)
5. Регистрация HTTP-маршрутов и запуск веб-сервера

### Основной цикл

- Проверка подключения к WiFi — при потере связи МК перезагружается (`ESP.restart()`)
- Обработка входящих HTTP-запросов через `server.handleClient()`

## HTTP API

### `GET /data`

Возвращает текущие показания датчика.

**Успешный ответ** (`200 OK`):
```json
{
  "room": "Bedroom",
  "temp": 21.5,
  "humidity": 48.2
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `room` | string | Название комнаты из `config.h` |
| `temp` | float | Температура в °C (1 знак после запятой) |
| `humidity` | float | Относительная влажность в % (1 знак после запятой) |

**Ответ с ошибкой** (`500 Internal Server Error`):
```json
{ "error": "temp sensor read failed" }
```
```json
{ "error": "humidity sensor read failed" }
```

Возвращается, когда DHT22 выдаёт `NaN`.

### `GET /ping`

Эндпоинт проверки доступности.

**Ответ** (`200 OK`):
```
OK
```

### `GET /`

Псевдоним для `GET /data`.

## Конфигурация

Вся настройка находится в [`include/config.h`](include/config.h). Перед прошивкой каждого узла необходимо задать индивидуальные значения:

| Параметр | Описание | Пример |
|----------|----------|--------|
| `WIFI_SSID` | Имя WiFi-сети | `"TP-Link_38CA"` |
| `WIFI_PASSWORD` | Пароль WiFi | `"07061257"` |
| `ROOM_NAME` | Идентификатор комнаты (в JSON-ответе) | `"Bedroom"` |
| `STATIC_IP` | Статический IP-адрес узла | `192.168.0.101` |
| `GATEWAY` | Шлюз сети | `192.168.0.1` |
| `SUBNET` | Маска подсети | `255.255.255.0` |
| `DHT_PIN` | GPIO-пин подключения линии данных DHT22 | `D6` |

Альтернативная конфигурация для другой локации: [`include/config.krasnoadar.h`](include/config.krasnoadar.h).

## Сборка и прошивка

Требуется [PlatformIO](https://platformio.org/).

```bash
# Сборка
pio run

# Прошивка (подключайте по одной D1 Mini)
pio run --target upload

# Монитор последовательного порта
pio device monitor
```

В `platformio.ini` используются флаги `--no-stub` и `--before=no_reset` для совместимости с платами D1 Mini, имеющими нестабильное USB-Serial соединение.
