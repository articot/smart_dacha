# Smart Dacha — Мониторинг температуры и управление отоплением

Полностью Arduino-based система умного дома для мониторинга температуры и влажности в нескольких комнатах и автоматического управления газовым котлом. Разработана с помощью PlatformIO в VS Code.

---

## Архитектура

```
Браузер / Телефон
      │  HTTP GET /
      ▼
  [ESP32 Контроллер]  ──── GPIO 26 ────▶  Реле ──▶  Газовый котёл
  192.168.1.100
  + SIM800L GSM
      │
      ├─ GET /data ──▶  [D1 Mini — Спальня]      192.168.1.101
      ├─ GET /data ──▶  [D1 Mini — Кухня]         192.168.1.102
      └─ GET /data ──▶  [D1 Mini — Гостиная]      192.168.1.103
```

## Компоненты

| Кол-во | Компонент | Назначение |
|--------|-----------|------------|
| 3×  | Wemos D1 Mini (ESP8266) | Узел датчика — по одному на комнату |
| 3×  | DHT22 + резистор 10 кОм | Датчик температуры и влажности |
| 1×  | ESP32 DevKit | Контроллер, веб-панель, логика управления |
| 1×  | GSM-модуль SIM800L | SMS-уведомления и дистанционное управление |
| 1×  | Модуль реле 5 В (1 канал) | Коммутация газового котла |
| —   | Блоки питания 5 В (USB) | Питание всех плат |

### Подключение SIM800L → ESP32

| SIM800L | ESP32 |
|---------|-------|
| TX | GPIO 16 (RX2) |
| RX | GPIO 17 (TX2) |
| VCC | 3,7–4,2 В (отдельный источник питания) |
| GND | GND |

> **Важно:** SIM800L потребляет до 2 А в пиковые моменты. **Не** питайте его от вывода 3,3 В ESP32 — используйте отдельный аккумулятор LiPo или качественный источник питания 4 В.

### Подключение реле → ESP32

| Модуль реле | ESP32 |
|-------------|-------|
| IN | GPIO 26 |
| VCC | 5 В |
| GND | GND |

## Настройка

### 1. WiFi (оба проекта)

Отредактируйте `include/config.h` в **обоих** проектах — `controller/` и `node/`:

```cpp
#define WIFI_SSID     "НазваниеСети"
#define WIFI_PASSWORD "ПарольСети"
```

### 2. Узлы датчиков — прошивать каждый D1 Mini отдельно

Перед каждой прошивкой измените `node/include/config.h`:

| Узел | `ROOM_NAME` | `STATIC_IP` |
|------|-------------|-------------|
| 1 | `"Bedroom"` | `192.168.1.101` |
| 2 | `"Kitchen"` | `192.168.1.102` |
| 3 | `"Living Room"` | `192.168.1.103` |

### 3. Контроллер — `controller/include/config.h`

```cpp
#define ADMIN_PHONE     "+79991234567"  // Только с этого номера принимаются команды и на него приходят уведомления

#define TEMP_THRESHOLD  20.0f           // °C — котёл включается при падении ниже этого значения
#define HYSTERESIS       0.5f           // °C — котёл выключается при превышении порога + гистерезис

#define RELAY_PIN       26
#define RELAY_ON        HIGH            // Измените на LOW, если ваше реле активируется низким уровнем
```

### 4. Роутер — назначение статических IP-адресов

В настройках DHCP роутера зарезервируйте IP-адреса по MAC-адресам:

| Устройство | IP-адрес |
|------------|----------|
| ESP32 (контроллер) | `192.168.1.100` |
| D1 Mini #1 (Спальня) | `192.168.1.101` |
| D1 Mini #2 (Кухня) | `192.168.1.102` |
| D1 Mini #3 (Гостиная) | `192.168.1.103` |

## Сборка и прошивка

Откройте папку `controller/` или `node/` в VS Code с установленным PlatformIO.

```
Боковая панель PlatformIO → Build   (Ctrl+Alt+B)
Боковая панель PlatformIO → Upload  (Ctrl+Alt+U)
```

Или через терминал:
```bash
cd controller
pio run --target upload

cd ../node
pio run --target upload
```

## Веб-панель

Откройте `http://192.168.1.100` в любом браузере в той же сети.

- **Карточки комнат** — текущая температура (°C) и влажность (% RH); затемняются, если узел недоступен
- **Панель котла**
  - Режим **Авто** — котёл управляется автоматически по температурным порогам
  - Режим **Ручной** — появляются кнопки «Включить» / «Выключить»
- Страница автоматически обновляется каждые 10 секунд

**JSON API:** `GET http://192.168.1.100/api/status`

```json
{
  "heater": true,
  "mode": "auto",
  "rooms": [
    { "name": "Bedroom",     "temp": 19.8, "humidity": 47.0, "online": true },
    { "name": "Kitchen",     "temp": 21.2, "humidity": 55.1, "online": true },
    { "name": "Living Room", "temp": 20.5, "humidity": 50.3, "online": true }
  ]
}
```

## Управление по SMS

Отправьте SMS с номера `ADMIN_PHONE` на SIM-карту, вставленную в SIM800L:

| Сообщение | Действие |
|-----------|----------|
| `STATUS` | Ответ с температурой/влажностью всех комнат + состояние и режим котла |
| `HEATER ON` | Принудительно включить котёл — ответ с подтверждением |
| `HEATER OFF` | Принудительно выключить котёл — ответ с подтверждением |
| `HEATER AUTO` | Включить автоматический режим — ответ с подтверждением |

Команды с любых других номеров молча игнорируются.

## Автоматические SMS-уведомления

Контроллер автоматически отправляет уведомления на `ADMIN_PHONE`:

| Событие | Сообщение |
|---------|-----------|
| Температура в любой комнате опускается **ниже** `TEMP_THRESHOLD` | `ALERT: Temp dropped below 20.0C!` + показания всех комнат + `Heater: ON (Auto)` |
| Температура во всех комнатах восстанавливается **выше** `TEMP_THRESHOLD + HYSTERESIS` | `INFO: Temp restored above 20.5C.` + показания всех комнат + `Heater: OFF (Auto)` |

Уведомления отправляются только при **смене состояния** — не при каждом опросе датчиков.
